Connecting With External Firefox Instance from Robot Framework

This worked at least in May 2025 ... Additional arguments when starting Firefox: --marionette --remote-allow-hosts localhost

Robot keyword to establish a connection with Firefox:

*** Keywords ***
Connect With Existing Firefox
    ${service} =    Evaluate    selenium.webdriver.FirefoxService(log_output='geckodriver.log', service_args=['--connect-existing','--marionette-port','2828','--log','debug'])
    ${webdriver} =    Create Webdriver    Firefox    service=${service}
    RETURN    ${webdriver}

This roughly corresponds to calling Open Browser keyword from SeleniumLibrary and the browser can be used from Robot Tests. However, I could not get Close Browser keyword to work. This can be solved using some native library like AutoIT for closing the browser window.
